7.1逆序对计数问题的推广

使用归并排序对数组进行排序，排序过程中需要合并左右两个子数组，并在合并的过程中统计逆序对数量。

在合并过程中，我们使用三个指针i、j、k分别指向左子数组、右子数组和合并后的数组。如果左子数组中的元素大于右子数组中的元素的C倍，即A[i] > C * A[j]，那么说明右子数组中的这个元素可以和左子数组中i到mid之间的所有元素组成逆序对，因此需要将右子数组中的这个元素放在合并后的数组中，并将count加上mid-i+1。如果左子数组中的元素小于等于右子数组中的元素的C倍，那么说明这个元素不会和右子数组中的任何元素组成逆序对，直接将它放在合并后的数组中即可。

7.4

(1)将前两个数组合并的时间复杂度为O(n)，合并后的数组中有2n个元素，将已经合并的数组与第三个数组合并的时间复杂度为O(2n + n) = O(3n)，依次类推，将已经合并的数组与第k个数组合并的时间复杂度为O((k-1)n + n) = O(kn)，合并后的数组中有kn个元素。

因此，总的时间复杂度为:

O(n + 2n + 3n + ... + kn) = O(n(1 + 2 + 3 + ... + k)) = O(nk(k+1)/2) = O(nk2)

(2)记该算法为merge，将k个元素的数组分成两个规模为k/2的子问题，对这两个子问题分别递归地调用merge。 对于每一次递归，我们需要将两个排好序的子数组合并成一个排好序的数组，这个操作可以在O(kn)的时间内完成。

T(k)=2T(k/2)+O(kn) 由Master定理复杂度为O(nklogk)

14.2堆中第k大的元素

因为堆中第k大的元素最多在第k层(反证法)，所以可以把k+1层以下的删除，再通过popk次找出第k大的元素

14.3d叉堆

\[ \begin{array} 假设i在第t层第k个,i=\frac{d^{t-1}-1}{d-1}+k\\ 第j个子节点为\frac{d^{t-1}-1}{d-1}+k+d^{t-1}-k+d(k-1)+j=d(i-1)+j+1\\ d(i-1)+j+1 \in [d(i-1)+2，d(i-1)+d+1]\\ 所以i节点的父节点为\lfloor\frac{i-2}{d}+1\rfloor证毕 \end{array} \]

14.4

\[ 高度为h的完美堆的所有节点高度和为1\cdot2^{h-1}+2\cdot2^{h-2}+3\cdot2^{h-3}...h\cdot2^0=2^{h+1}-2-h \]

数学归纳法 $$ \[\begin{array} \ 1. 当n=1时，高度为0，结论显然成立\\ 2. 假设n<=k时命题均成立，那么对于n=k+1，设堆的高度为h,\\根据堆的特性可知，其左子堆和右子堆至少有一个是完美二叉树\\ (1)若整个堆是完美二叉树\\ 则高度和为2^{h+1}-h-2=n-1-h<n-1,结论成立\\ (2)若左子堆非完美，右子堆完美，则左子堆高度为h-1，右子堆高度为h-2\\ 左子堆 N=(n-1)-(2^{h-1}-1)=n-2^{h-1}\\ 整个堆的高度和\le n-2^{h-1}-1+h+2^{h-1}-h=n-1，结论成立\\ (3)若左子堆完美，右子堆非完美，则左子堆高度为h-1，右子堆高度为h-1\\ 右子堆 N=(n-1)-(2^{h}-1)=n-2^{h}\\ 整个堆的高度和\le 2^{h}-h-1+h+n-2^{h}-1=n-2<n-1，结论成立\\ (4)左右子堆都是完美堆，但本身不是完美堆，则左子堆高度为h-1，右子堆高度为h-2\\ 整个对的高度和为2^{h-1}-h+2^{h}-h-1+h=2^{h+1}-2^{h-1}-h-1=n-h<n-1,结论成立\\ 3. 综上所述一个有n个节点的堆中，所有节点的高度之和最多为n-1\\且当堆去掉一个节点为完美⼆叉树时，则其高度之和正好是n-1 \end{array}\]

$$

14.5

将链表分成两个部分，对每个部分进行递归操作，然后将两个部分合并成一个有序链表。每次归并操作需要O(n)的时间复杂度。因为每次归并操作后链表的长度减半，所以总共需要进行logk次归并操作，因此总时间复杂度为O(n log k)

14.6动态发现中值

通过维护两个堆来解决，一个最小堆和一个最大堆，最小堆存储较大的一半元素，最大堆存储较小的一半元素。这样，中位数就可以在这两个堆的堆顶元素之间找到。
